 How to Run the RAG Chatbot Project Successfully

  Here are the step-by-step instructions to run the project:

  Prerequisites

  - Python 3.8+ installed
  - Node.js and npm installed (for Docusaurus frontend)

  Step 1: Backend Setup

  1. Navigate to the backend directory:
  cd backend
  2. Set up Python virtual environment (if not already done):
  python -m venv venv
  venv\Scripts\activate  # On Windows
  # source venv/bin/activate  # On macOS/Linux
  3. Install Python dependencies:
  pip install fastapi==0.104.1 uvicorn[standard]==0.24.0 openai==1.3.5 qdrant-client==1.7.0 psycopg2-binary==2.9.9 python-dotenv==1.0.0 pydantic==2.5.0 pydantic-settings==2.1.0 asyncpg==0.29.0 sqlalchemy==2.0.23 alembic==1.13.1 pytest==7.4.3 pytest-asyncio==0.21.1 httpx==0.25.2
  4. Start the backend server:
  python -m uvicorn src.main:app --reload

  4. The backend will start on http://localhost:8000

  Step 2: Frontend Setup (Docusaurus)

  1. Navigate to the book directory:
  cd book  # From the project root, or from backend: cd ../book
  2. Install Node.js dependencies:
  npm install
  3. Start the Docusaurus development server:
  npm start

  3. The frontend will start on http://localhost:3000

  Step 3: Using the Chatbot

  1. Access the application:
    - Open your browser and go to http://localhost:3000
    - You should see the Docusaurus site with the RAG chatbot
  2. Using the chatbot:
    - Look for the chatbot icon (speech bubble) in the bottom-right corner
    - Click it to open the chat interface
    - Type your questions about the Physical AI & Humanoid Robotics textbook
    - The chatbot will respond using RAG (Retrieval Augmented Generation) based on the textbook content

  Troubleshooting Tips

  1. If you get port conflicts:
    - Backend: If port 8000 is in use, use --port 8001 with uvicorn
    - Frontend: If port 3000 is in use, Docusaurus will suggest an alternative port
  2. If you encounter database issues:
    - Make sure the textbook_rag.db file exists in the backend directory
    - You may need to run python init_db.py in the backend directory if the database is missing
  3. If the chatbot doesn't appear:
    - Check that the RagChatbot.jsx component is properly imported in your Docusaurus pages
    - Verify that the backend server is running on http://localhost:8000
  4. Environment variables:
    - Create a .env file in the backend directory if needed with API keys for OpenAI, Qdrant, etc.

  Project Structure Overview

  - backend/ - Contains the FastAPI backend server
  - book/ - Contains the Docusaurus frontend
  - The chatbot component is at book/src/components/RagChatbot.jsx

  The backend handles the RAG functionality and chat history, while the frontend provides the user interface and integrates with the backend API.
